# 1. Requirements

## 1.1 Functional Requirements
* The system shall allow the user to look for the resources or textbooks they want via a search engine function.
* The system shall allow the user to borrow a selected book by inputting their full name, faculty ID, and the required borrowing duration.
* It should also provide a dashboard for librarians to catalog and organize the books based on the resources content such as subjects, categories or fields of study.
* The system shall notify the borrower of any fees assessed if a returned book or library resource is marked as damaged.

## 1.2 Non-Functional Requirements
* The system shall process user search, book selection, and borrowing requests with a response time not exceeding 3 seconds.
* The system shall securely store all user’s passwords and private information using high standard encryption.
* The system shall guarantee an allowable downtime of maximum 30 minutes per day.
* The system shall be cross platform compatible, functioning on standard web browsers, Android devices, and major operating systems (Windows, macOS, and Linux).
